# v0.2 桌面体验版设计

> 日期：2026-07-07
> 范围：托盘运行、关闭最小化、开机自启、日志保留、配置目录管理、导入导出
> 目标：把 FRP Manager 从"能用的 GUI"提升为"像样的桌面应用"

## 背景

v0.1 已完成核心闭环（服务端 agent + frps 内嵌、客户端 GUI + frpc 内嵌），并已迁移到 Wails v3（`v3.0.0-alpha2.115`）。Wails v3 提供了稳定的 `app.SystemTray.New()`、`RegisterHook(events.Common.WindowClosing)`、`WindowsOptions.DisableQuitOnLastWindowClosed` 等托盘相关 API，托盘方案不再有阻塞风险。

`SettingsView.vue` 当前只有静态骨架（开关和输入框无逻辑），本次一并接线。

## 架构总览

新增三个客户端内部包：

| 包 | 职责 |
|----|------|
| `client/internal/settings` | 应用设置读写（JSON 文件，存配置目录下 `settings.json`） |
| `client/internal/logfile` | 日志持久化（按天轮转，保留 N 天） |
| `client/internal/autostart` | 开机自启（Windows 注册表 / Linux .desktop / macOS LaunchAgent） |

新增 `client/tray.go`：托盘 + 窗口生命周期绑定逻辑，保持 `main.go` 简洁。

设置数据流：前端设置页 ↔ `App.GetSettings/SaveSettings` ↔ `settings.Store` ↔ `settings.json`。日志数据流：`App.EmitLog` → `logfile.Writer`（落盘）+ `app.Event.Emit("log:append")`（推前端）。开机自启：前端开关 ↔ `App.SetAutoStart` ↔ `autostart.Enable/Disable`。

## 详细设计

### 1. 设置持久化（settings 包）

**文件**：`client/internal/settings/settings.go`

```go
type Settings struct {
    CloseToTray      bool `json:"close_to_tray"`      // 关闭时最小化到托盘
    AutoStart        bool `json:"auto_start"`         // 开机自启
    LogRetentionDays int  `json:"log_retention_days"` // 日志保留天数，0=不落盘
    ConfigDir        string `json:"config_dir"`       // 空则用默认目录（只读展示，不可改）
}

type Store struct { path string }
func NewStore(path string) *Store
func (s *Store) Load() (Settings, error)         // 文件不存在返回零值+nil
func (s *Store) Save(Settings) error             // 原子写（写临时文件再 rename）
```

默认路径：`config.DefaultDir()` + `/settings.json`。零值约定：`LogRetentionDays=0` 表示不落盘（与现有"日志只在内存"行为一致，向后兼容）。

### 2. 托盘 + 关闭最小化（tray.go）

**文件**：`client/tray.go`

```go
type Tray struct {
    app     *application.App
    window  *application.WebviewWindow
    systray *application.SystemTray
    icon    []byte
}

func NewTray(app *application.App, window *application.WebviewWindow, icon []byte) *Tray
func (t *Tray) Setup()                  // 创建托盘 + 菜单 + 注册 WindowClosing hook
func (t *Tray) ShowWindow()             // 显示并聚焦主窗口
func (t *Tray) Quit()                   // 真正退出：标记 quitting 后 app.Quit()
```

**菜单项**：显示主窗口、启动全部 frpc（TODO v0.3 再做，本次只放"显示"和"退出"）、分隔符、退出。

**关闭行为**：`window.RegisterHook(events.Common.WindowClosing, ...)`：
- 读取 `settings.CloseToTray`：true → `window.Hide()` + `e.Cancel()`；false → 不取消，正常退出。
- 退出菜单/托盘"退出"项：设置 `quitting=true` 标志后调 `app.Quit()`，并确保 `ServiceShutdown` 停止所有 frpc。

**app.Options** 增加：
```go
Windows: application.WindowsOptions{
    DisableQuitOnLastWindowClosed: true,  // 关闭窗口不退出应用
},
```

**图标**：复用 `client/build/appicon.png`（embed）。若加载失败用空字节（Windows 显示默认图标）。

### 3. 开机自启（autostart 包）

**文件**：`client/internal/autostart/autostart.go`

按平台分文件：
- `autostart_windows.go`：注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`，键名 `FRPManager`，值为当前可执行文件路径。
- `autostart_linux.go`：`~/.config/autostart/frp-manager.desktop`，Exec=当前 exe。
- `autostart_darwin.go`：`~/Library/LaunchAgents/com.kdc.frp-manager.plist`（本次仅占位实现，用户主要在 Windows）。

统一接口：
```go
func Enable() error   // 写注册表/文件
func Disable() error  // 删除
func IsEnabled() bool // 检查是否存在
```

### 4. 日志保留（logfile 包）

**文件**：`client/internal/logfile/logfile.go`

```go
type Writer struct {
    dir          string
    retentionDays int
    mu           sync.Mutex
    currentFile  *os.File
    currentDate  string
}
func New(dir string, retentionDays int) *Writer
func (w *Writer) Write(level, message, serverID string) error  // 按天分文件 logs/YYYY-MM-DD.log
func (w *Writer) Close() error
func (w *Writer) Cleanup() error  // 删除超过 retentionDays 的日志文件
```

行为：
- `retentionDays <= 0` 时 `Writer` 为 no-op（所有方法直接返回 nil）。
- 每条日志追加写入 `{dir}/logs/{YYYY-MM-DD}.log`，格式 `RFC3339 [level] [serverID] message`。
- 日期变更时切换文件。`Cleanup()` 启动时调用一次 + 每天定时调用（用 `time.Ticker`）。

**集成点**：`App.ServiceStartup` 加载 settings 后，若 `LogRetentionDays > 0` 则创建 `logfile.Writer`；`EmitLog` 同时写文件和推事件；`ServiceShutdown` 关闭 writer。

### 5. 配置目录管理

**只读展示**：`App.GetConfigDir()` 返回当前 `config.DefaultDir()` 路径供设置页显示。修改目录涉及数据迁移（数据库 + 日志 + settings），v0.2 范围内不做迁移，仅展示路径 + 提供"打开目录"按钮（前端用 `window.open` 或后端 `open` 命令）。

**打开目录**：新增 `App.OpenConfigDir()`，Windows 用 `exec.Command("explorer", dir)`，跨平台用 `exec.Command("xdg-open"/"open")`。

### 6. 导入导出

**文件**：`client/internal/backup/backup.go`（或直接放 app.go，数据量小，倾向后者）

```go
type Backup struct {
    Servers []db.Server `json:"servers"`
    Tunnels []db.Tunnel  `json:"tunnels"`
    ExportedAt time.Time `json:"exported_at"`
    Version    string    `json:"version"`  // "0.2"
}
func (a *App) ExportData() (string, error)  // 返回 JSON 字符串，前端触发下载
func (a *App) ImportData(json string) error // 解析后清空并重插 servers/tunnels
```

前端：设置页"导出"按钮调 `api.exportData()`，拿到 JSON 字符串后用 Blob + `<a download>` 触发下载；"导入"用 `<input type=file>` 读取文件内容调 `api.importData(content)`。

导入策略：**全量替换**（先删后插），简单可靠。导入时不调用 agent 校验/注册（用户可能离线导入），仅落库；用户后续手动启动 frpc 时若端口/域名冲突会在 agent 校验阶段报错。

### 7. 前端设置页（SettingsView.vue）

绑定后端方法，所有开关/输入实时保存（change 事件触发 `api.saveSettings`）。结构：
- 开关：关闭时最小化、开机自启
- 数字：日志保留天数（0-30，0=不落盘）
- 只读：配置目录路径 + "打开目录"按钮
- 按钮：导出数据、导入数据（file input）
- 保存按钮（全量保存，兜底）

新增 store：`stores/settings.ts`，存当前 settings 并提供 `load/save`。

### 8. App 新增方法汇总

挂到 `App` 上供前端绑定调用：
- `GetSettings() (Settings, error)`
- `SaveSettings(s Settings) error`
- `GetConfigDir() string`
- `OpenConfigDir() error`
- `ExportData() (string, error)`
- `ImportData(json string) error`

内部方法：`SetTray(*Tray)`（main.go 回填，供后续扩展）。

## 风险与边界

1. **托盘图标 embed**：`build/appicon.png` 已存在；若 Wails 对 PNG 尺寸有要求（16/32），可能需额外生成小图标。先用现有图标，运行验证。
2. **导入全量替换风险**：用户误点导入会清空现有数据。前端导入前加 `ElMessageBox.confirm` 二次确认。
3. **开机自启权限**：Windows HKCU 无需管理员权限；Linux/macOS 同样是用户级。无 UAC 弹窗。
4. **日志文件并发**：单进程内 `sync.Mutex` 已足够；多实例已被单实例锁阻止。
5. **ConfigDir 修改未实现**：设置页只展示，文案明确写"（只读，修改需迁移数据，暂不支持）"。

## 范围边界（本次不做）

- 托盘"启动全部 frpc"菜单项（v0.3 服务端增强时一并做状态联动）
- frps 日志读取、端口池动态编辑（v0.3）
- stcp/xtcp 等高级协议（v0.4）
- 自动更新、安装包（v1.0）
